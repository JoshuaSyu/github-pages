要把檔案恢復到編輯前的狀態，大家都是怎麼做的呢？

最簡單的方法就是先複製編輯前的檔案，使用這個方法時通常都會在檔案名稱或目錄名稱上添加編輯的日期。

那多人編輯同一個檔案，會發生什麼事情呢?

先進行編輯的人的修改內容會被覆蓋，相信大家都有這樣的經歷。

版本控制

Git、SVK、SVC、BitKeeper、Perforce、Subversion...

什麼是版本控制?

版本控制是維護工程藍圖的標準作法，能追蹤工程藍圖從誕生一直到定案的過程。此外，版本控制也是一種軟體工程技巧，藉此能在軟體開發的過程中，確保由不同人所編輯的同一程式檔案都得到同步。

版本控制其實是...

可以回到過去的任何一個時間點。

版本控制也是...

善用 branch 可降低多人開發會遇到的問題

總之，版本控制可以告訴我們

  • 為什麼變更
  • 什麼時候變更
  • 變更了什麼
  • 是誰變更的
  • 切換到過去進度
  • 切換不同版本
  • 合併不同版本

多人開發使用版本控制後

當有人想覆蓋其他人的最新檔案時，系統會發出警告，因此可以避免在無意中覆蓋他人的編輯內容。

Git 基礎概念

遠端數據庫和本地端數據庫

  • 遠端數據庫：配有專用的伺服器 ( github )，為了讓多人共享而建立的數據庫 ( repository )。
  • 本地端數據庫：為了方便用戶個人使用，在自己的機器上配置的數據庫 ( repository )。

Git 基礎指令

初始化


git init

幫你在專案身成所需要得檔案 .git，專案只要執行一次即可

設定身份


git config --global user.name "名字"
git config --global user.email "email"
git config --list // 命令列出所有Git能找到的設定值
git config --unset-all user.name // 清除 user.name

若使用 --global，所有的 git project 都會採用此資訊

檢視 Git 當前狀態


git status

加到暫存區


git add

送進 repository


git commit
git commit -m "commit message"

記錄


git log

分支


git branch // 顯示本機分支列表
git branch -r // 顯示遠端 repository 分支列表
git branch [new branch] // 新增分支
git branch -D [new branch] // 刪除分支

切換


git checkout [branch name | commit hash]

合併


git merge [branch name] // 將 branch name 合併到目前分支上

遠端


git remote add [remote name] git@HOSTPATH // 新增遠端
git remote -v // 顯示遠端資訊
git remote rm [remote name] // 刪除遠端

複製遠端 repository


git clone git@HOSTPATH:directory name

更新至遠端


git push [remote name] [branch name]

更新本地端


git pull [remote name] [branch name]

Git 起步走


1. cd project path
2. git init
3. git add index.html
4. git commit -m "first commit"
